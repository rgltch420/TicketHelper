# Ticket Management System

## Description

TicketHelper is a Single Page Application (SPA) developed with JavaScript modules. The system allows users to manage technical support tickets through different roles: Administrator, Technician, and Client.

The application uses two independent JSON Server services. One service handles authentication and users, while the other manages ticket information.

The project was developed as a learning exercise focused on modular architecture, role-based access control, API consumption, and CRUD operations.

## Installation

1. Clone or download the project.

2. Install dependencies:

```bash
npm install
```

3. Make sure the following files are available:

```text
auth-db.json
data-ticket.json
```

---

## How to Run the Project

### 1. Start the Authentication Service

Open a new terminal and run:

```bash
npx json-server --watch auth-db.json --port 3001
```

### 2. Start the Ticket Service

Open another terminal and run:

```bash
npx json-server --watch data-ticket.json --port 3002
```

### 3. Start the Application

Open a third terminal and run:

```bash
npm run dev
```

### 4. Open the Application

Open the local URL generated by Vite in your browser.

### Start the Application

Open a third terminal and run:

```bash
npm run dev
```

Then open the local URL generated by Vite.

## Role Behavior

### Administrator

The administrator has full access to the system.

Features:

* View all tickets
* Create tickets
* Edit tickets
* Delete tickets
* Assign technicians
* Assign ticket status
* View ticket information

### Technician

The technician manages assigned tickets.

Features:

* Create tickets
* View assigned tickets
* Edit assigned tickets
* Update ticket status

### Client

The client reports issues and requests support.

Features:

* Register in the system
* Create tickets
* View personal tickets
* Edit tickets according to business rules

---

## Authentication and Authorization

The application includes:

* Login system
* User registration
* Session persistence using Local Storage
* Protected routes
* Role-based access control
* Logout functionality

Only authenticated users can access protected views.

---

## Ticket Management

Each ticket contains:

* Ticket name
* Case type
* Description
* Assigned technician
* Client information
* Status

Available statuses:

* Pending
* Assigned
* In Progress
* Solved

A status can only be assigned when a technician is selected.

---

## Technical Decisions

The following decisions were made during development:

* JavaScript modules were used to improve code organization.
* Services were separated from views and pages.
* JSON Server was used to simulate backend APIs.
* Local Storage was used to maintain user sessions.
* SPA navigation was implemented using Hash Routing.
* CRUD operations were separated into reusable service functions.
* Role validation was implemented inside the router to protect routes.
* Two independent APIs were used to separate authentication data from business data.
